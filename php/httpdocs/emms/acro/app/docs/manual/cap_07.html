MANUAL DEL USUARIO

7. Operaciones con préstamos

7.1 Buscar y ver datos de un préstamo

7.1.1 Búsqueda del préstamo a nivel de cliente

Esta forma permite buscar una ficha de préstamo (a nivel de cliente) independientemente del estado del mismo - saldado o no, etc. La forma de búsqueda le permite encontrar rapidamente la ficha del préstamo con el uso de filtros opcionales. Para iniciar la búsqueda debe ir a:

MENU: Operaciones cartera > Buscar préstamos

Criterios de busqueda:

Código del préstamo
Es una cadena alfanumérica de X caracteres que funciona como un identificador único para cada préstamo. Nótese que aún en los préstamos otorgados a grupos, cada miembro del grupo es tratado por el sistema como un beneficiario individual asignándole un código único de préstamo.
1er Nombre & Apellidos
Estos campos permiten buscar al cliente asociado al préstamo entrando su nombre o apellido, nótese que se aceptan cadenas incompletas. Por ejemplo, para buscar los préstamos asociados a una persona de apellido Alcantara, basta con introducir las primeras letras ie (alc) en el campo de apellido.
Estado
Presenta una lista para seleccionar entre todos los estados posibles en que puede estar un préstamo.
Sucursal
Permite elegir entre cualquiera de las sucursales registradas en la base de datos, útil para limitar los resultados de la búsqueda, evitando generar una lista demasiado larga.
Tipo
Permite especificar el tipo de asociado beneficiario del préstamo. Normalmente solo existen 2 categorías: Individual o Grupal.
Asesor
Permite elegir entre cualquiera de los asesores registrados en la base de datos, útil para limitar los resultados de la búsqueda, evitando generar una lista demasiado larga.

Lista de Resultados

Al enviar el formulario el sistema presenta una lista de resultados con los siguientes datos:

  • Código del préstamo
  • Estado del préstamo
  • Nombre del cliente beneficiario

La lista de resultados contiene también los siguientes hipervínculos:

Ver
Para entrar directamente en la ficha del préstamo.
Editar
Para entrar directamente en la forma de edición del préstamo, en caso de que el estado del mismo lo permita.

Para ver los datos del préstamo haga clic convenientemente en el hipervínculo 'Ver'.

7.1.2 Búsqueda del préstamo a nivel de grupo

Para localizar la vista de con la Información general del préstamo, la cual contiene la información del préstamo a nivel de grupo, siga las instrucciones que aparecen en el capítulo 7.6 Buscar cheques.

7.2 Aplicar pagos a nivel de cliente

Esta opción permite aplicar pagos en las circunstancias más diversas, siempre que el préstamo se encuentre en estado 'Entregado'. Lo mismo avances al principal, cancelación total de la deuda, que el pago de cuotas atrasadas.

Sirve lo mismo para clientes individuales que para miembros de grupos, sin embargo recomendamos que para miembros de grupos que esten al dia con sus pagos, se proceda con la opción de pago a nivel de grupo. Para más detalles vea el capítulo 7.5 Aplicar pagos a nivel de grupo. Para préstamos individuales, lo mismo con cuotas vencidas sin atraso o con atraso, recomendamos proceder como se indica en el capítulo 7.3 Aplicar pago de cuota vencida o en atraso.

La aplicación de los pagos tiene una restricción importante, no se aceptan pagos de fracción de una cuota atrasada, la cuota debe ser pagada en su totalidad. Sin embargo, a un préstamo con varias cuotas atrasadas, se le acepta el pago de las cuotas más antiguas, una a la vez; es decir, no tiene que pagar necesariamente el total de todas las cuotas atrasadas.

Para poder realizar cualquier tipo de pago a nivel de cliente, debe ir a la ficha de datos del préstamo correspondiente. Para ubicar la ficha siga las instrucciones del capítulo 7.1.1 Búsqueda del préstamo a nivel de cliente. Para ahorrarse tiempo en la búsqueda asegúrese de filtrar solo para préstamos cuyo estado sea 'Entregado'.

Una vez en la ficha, si el préstamo esta en estado 'Entregado', podrá localizar en la barra de íconos ubicada al final de la ficha de datos, las siguientes opciones:

Aplicar pago
Seleccionar esta opción para aplicar el pago de la cuota correspondiente.

Si el préstamo tiene cuotas atrasadas, permitirá pagar el total de los atrasos o el monto correspondiente a la cuota atrasada más antigua, incluyendo las penalidades asociadas a dicha cuota.

Si el préstamo no tiene cuotas pendientes, permitirá escojer entre las siguientes opciones:
  • Pagar la próxima cuota por adelantado.
  • Hacer un pago de avance al principal.
  • Aplicar pago completo - Con esto se salda el préstamo.

En todos los casos, el sistema presenta los montos máximos y mínimos permitidos y no va a aceptar montos fuera de dicho rango. En caso necesario, por ejemplo un pago por adelantado, etc., el calendario se ajusta convenientemente de manera automática, respetando los términos originales del contrato del préstamo, pero haciendo los ajustes de lugar para no afectar al cliente con el cálculo de intereses, etc.
Aplicar pago completo
Seleccionar esta opción para aplicar el pago total de todas las cuotas pendientes, incluyendo penalidades y balance de princpial completo. Con esto se salda el préstamo.

7.3 Aplicar pago de cuota vencida o en atraso

Esta función permite aplicar pagos a préstamos con cuotas vencidas, lo mismo al dia que atrasadas. Presenta una forma de búsqueda con filtros opcionales para localizar los préstamos rapidamente. Solo se presentan resultados a nivel de cliente, o sea, en caso de grupos, se presenta un registro para cada cliente del grupo con cuotas vencidas o en atraso. Para aplicar el pago debe ir a:

MENU: Operaciones cartera > Ver pagos atrasados

Criterios de búsqueda

Sucursal
Seleccione de una lista de programas registrados para limitar los resultados de la búsqueda.
Asesor
Seleccione de una lista de oficiales de crédito registrados para limitar los resultados de la búsqueda.
Tipo de Préstamo
Seleccione entre Individuales o Grupos. Si se especifica "Grupos", se presentarán los resultados a nivel de cada cliente miembro del grupo.

Una vez el formulario de búsqueda es validado el sistema presentará una lista de resultados, con los datos siguientes:

  • Código
  • Cliente
  • Cuotas- No. cuotas atrasadas.
  • Mora - Total combinado de dias de atraso.

Desde la lista de resultados podrá ejecutar las siguientes opciones:

  • Ver ficha préstamo.
  • Aplicar pago a préstamo.

7.4 Creación de hoja para colectar pagos a nivel de grupo

El propósito de esta función es crear una hoja para colectar los pagos de un grupo. Nótese que no se trata de una función para aplicar pagos a nivel de grupo, sino que es una hoja de ayuda para que el asesor de créditos tenga los detalles de cada pago pendiente dentro del grupo. Generalmente el asesor debe imprimir esta hoja antes de ir a la reunión de colección de pagos con el grupo. Se puede imprimir la hoja con dias de antelación.

Esta hoja solo muestra los detalles de la próxima cuota grupal pendiente de pago en el calendario, no importa si la cuota esta atrasada o no. Sin embargo, aunque la hoja muestra la cantidad de dias en atraso (si los hubiera) al momento de imprimir la hoja, no presenta el cálculo de la penalidad asociada, de manera que es más efectiva para grupos que están al dia. Para saber el valor exacto a pagar en el dia corriente en caso de grupos con atraso, es necesario llamar la función de aplicar pagos de manera individual para cada cliente dentro del grupo. Para más detalles vea capítulo 7.2 Aplicar pagos a nivel de cliente. Para crear una hoja de pago debe ir a:

MENU: Operaciones cartera > Crear hoja de pagos

La forma de búsqueda le permite encontrar rapidamente al grupo para el cual se va a generar la hoja.

Criterios de búsqueda:

Código del Grupo
El código de grupo es una cadena alfanumérica única (ej. 030.B.00015) que identifica el grupo que se busca.
Nombre del Grupo
Se permite introducir nombres parciales, es decir si se busca un grupo llamado 'Mujeres Victoriosas' basta con introducir 'mujeres' para que salgan todos los grupos cuyos nombres incluyen la palabra 'mujeres', esto para el asesor o programa especificados en los restantes campos del formulario de búsqueda, que son campos obligatorios.
Asesor (*)
Presenta una lista de todos los asesores registrados en el sistema. Debe seleccionar uno para limitar el número de resultados
Programa (*)
Presenta una lista de todos los programas registrados en el sistema. Debe seleccionar uno para limitar el número de resultados

(*) Campos obligatorios

7.5 Aplicar pagos a nivel de grupo

Esta opción permite aplicar pagos a nivel de grupos, pero solo está disponible para grupos que están al dia con sus pagos. Para aplicar pagos a grupos con atraso, es necesario llamar la función de aplicar pagos de manera individual para cada cliente dentro del grupo. Para más detalles vea capítulo 7.2 Aplicar pagos a nivel de cliente. Para aplicar un pago grupal debe ir a:

MENU: Operaciones cartera > Aplicar pagos

Una forma de búsqueda permite encontrar rapidamente el grupo al cual se quiere aplicar pagos usando los siguientes filtros:

Criterios de búsqueda

Código grupo
Esta es la cadena alfanumérica única que identifica a cada grupo dentro del sistema. Es un patrón del tipo '021.B.03747'.
Nombre grupo
Este campo permite introducir una cadena parcial de caracteres como nombre de grupo, en caso de que no se tenga a mano el nombre exacto.
Asesor (*)
Presenta una lista de todos los asesores registrados en el sistema. Debe seleccionar uno para limitar el número de resultados
Programa (*)
Presenta una lista de todos los programas registrados en el sistema. Debe seleccionar uno para limitar el número de resultados

(*) Campos obligatorios

Lista de Resultados

Al enviar el formulario el sistema presenta una lista de resultados con los siguientes datos:

  • Nombre completo grupo
  • Código del Grupo (ej. 021.B.03747)
  • Mora (dias de atraso)

La lista de resultados contiene también los siguientes hipervínculos:

Ver
Para visualizar la ficha del grupo, con el listado de todos sus miembros.
Seleccionar
Selecciona el grupo específico al cual se le quiere aplicar el pago.

Ventana de aplicación del pago

Para aplicar el pago haga clic convenientemente en la opción 'Seleccionar' de la lista de resultados. Esto le presentará la ventana de aplicación del pago. Antes de confirmar el pago asegúrese de verificar que los montos son los correctos. Vea además que tiene la posibilidad, de incluir o no en el pago, a los clientes de manera selectiva. Si alguno de los clientes no pagó, pues se elimina de la lista de pagos y se procede con el resto.

IMPORTANTE: El sistema no acepta pago de cuotas parciales a nivel de cliente. Sin embargo, es factible aceptar que un grupo aplique las cuotas de los miembros que completaron y deje pendientes las de aquellos que no completaron, tal como se indica al final del párrafo anterior.

Si el monto pagado esta por encima de lo que indica la ventana de aplicación del pago, entonces el extra debe aplicarse individualmente a cada cliente según corresponda, como un avance al principal. Para más detalles vea capítulo correspondiente al pago con opción de avance al principal.

7.6 Buscar Cheques

Para iniciar la búsqueda de un cheque o solicitud de cheque, debe ir a:

MENU: Operaciones cartera > Préstamos y cheques > Buscar cheques

Criterios de búsqueda

No de Serie
Cadena alfanumérica única. Debe coincidir con el número de serie del cheque emitido en la fase de desembolso del préstamo. Es un campo opcional, asi permite realizar búsquedas de solicitudes de cheques aún pendientes de emisión.
Estado del cheque
Permite especificar el estado del cheque, puede ser:
  • Pendiente
  • Aprobado
  • Desembolsado
  • Entregado
Tipo de Beneficiario
Permite especificar el tipo de beneficiario(s) del cheque (grupos o individuales).
Sucursal
Presenta una lista de sucursales registradas en el sistema.
Solicitado por
Permite seleccionar el asesor que solicito el préstamo(s) a partir de una lista de usuarios registrados del sistema.
Desde... antes de
Permite especificar un rango de fechas definiendo un intervalo de tiempo en el cual se puede localizar cheque(s) con un estado o estatus determinado.

Lista de resultados

Presenta una tabla cuyas columnas / filas especifican el no. de serie del cheque, el nombre del asesor solicitante, y la sucursal y programa a que pertenecen los beneficiarios. Contiene hipervínculos que permiten ver los detalles de los préstamo(s) asociados al cheque.

7.7 Revertir pagos aplicados

Esta función permite revertir pagos aplicados por error. Esta acción elimina el pago, y restablece a su condición original anterior al pago erróneo, todos los elementos del préstamo incluyendo plan de pagos, historial, balances, etc. Sin embargo, el sistema deja una huella de cada reversión de pagos, incluyendo autor, fecha y código del préstamo. Existen además algunas restricciones que debe tener presentes:

  • Pagos aplicados el dia anterior o más viejos, no pueden ser revertidos
  • Solo puede revertir un pago el mismo usuario que lo aplica
  • Si se revierte el pago de un préstamo involucrado en un grupo, todos los pagos aplicados al grupo en el mismo día también serán revertidos.

Para revertir un pago debe ir a la página principal de entrada y hacer clic al icono:

INICIO: Ver pagos aplicados

Se presenta una ventana flotante que contiene un formulario de búsqueda, asegúrese de llenar los filtros convenientemente. Con el envío del formulario se obtiene una lista de resultados con todos los pagos aplicados que cumplen los criterios del filtro. Las lista incluirá una opción para revertir pagos si el usuario tiene los derechos necesarios.
